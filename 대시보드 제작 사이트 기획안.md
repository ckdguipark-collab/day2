# CV팀 DDD 대시보드 제작 사이트 기획안

## 프로젝트 개요

엑셀(CSV) 파일을 업로드하면 자동으로 영업 실적 대시보드를 생성하는 웹 애플리케이션.
전체 사업부부터 사업부별 → 병원별로 드릴다운하며 성분시장 마켓쉐어, 전달대비 성장률, 오리지널/제네릭 비율을 확인할 수 있다.

- **기술 스택**: Next.js 15 (App Router) + Supabase (PostgreSQL + Auth + Storage)
- **차트**: Recharts 또는 Chart.js
- **CSV 파싱**: Papa Parse (클라이언트)
- **스타일**: Tailwind CSS

## 실제 데이터 구조 분석

### raw 시트 (34,014행) — 핵심 데이터

| 컬럼 | 내용 | 예시 |
|------|------|------|
| `Brick Code` | 거래처 코드 | `90001` |
| `거래처명` | 병원명 | `힘찬병원` |
| `종합구분` | 병원 등급 | `종합` / `준종합` |
| `사업부` | 영업 사업부 | `병원1사업부` ~ `병원8사업부` |
| `팀` | 영업 팀 | `1팀`, `2팀`, `3팀` |
| `제품` | **자사 제품명** (비어있으면 경쟁사) | `리퀴시아`, `프리그렐`, `딜라트렌`, `베르쿠보` |
| `Pack Molecule String` | 성분시장 구분 + 제네릭 여부 | `RIVAROXABAN` (오리지널) / `RIVAROXABAN Gx` (제네릭) |
| `Product` | 제품 브랜드명 | `XARELTO`, `RIVAROXABAN Y.C` |
| `Manufacturer` | 제조사 | `BAYER KOREA`, `YUHAN CO.` |
| `Pack` | 규격 | `XARELTO C.TAB FILM 15MG 28` |
| `Jan 2024` ~ `Dec 2025` | 월별 DDD 처방량 | `180`, `90` ... |

### 브릭마스터 시트 (442행) — 병원/담당자 매핑

Brick Code → 병원명, 사업부, 팀, 담당자, 지역

### 자사 제품 4종

| 자사 제품 | 성분시장 |
|-----------|---------|
| 리퀴시아 | RIVAROXABAN |
| 프리그렐 | CLOPIDOGREL |
| 딜라트렌 | CARVEDILOL |
| 베르쿠보 | VERICIGUAT |

### 오리지널 vs 제네릭 판별

`Pack Molecule String`에 ` Gx` 포함 여부로 자동 판별
- `RIVAROXABAN` → 오리지널
- `RIVAROXABAN Gx` → 제네릭

## DB 스키마 (Supabase PostgreSQL)

```sql
-- 사용자 프로필
profiles (
  id uuid PK → auth.users.id,
  name text,
  role text  -- 'admin' | 'viewer'
)

-- 브릭마스터 (병원 마스터)
brick_master (
  brick_code text PK,
  hospital_name text,
  hospital_type text,     -- '종합' | '준종합'
  division text,          -- '병원1사업부' ...
  team text,
  employee_id integer,
  manager_name text,
  customer_code text,
  brick_name_k text       -- '서울 삼성서울병원'
)

-- 월별 DDD 실적 (long format으로 변환 저장)
monthly_ddd (
  id uuid PK,
  brick_code text FK → brick_master.brick_code,
  year_month date,              -- '2024-01-01' (월의 1일)
  ingredient_market text,       -- 'RIVAROXABAN' (Gx 제거)
  product_brand text,           -- 'XARELTO'
  manufacturer text,
  pack text,
  product_type text,            -- 'original' | 'generic'
  our_product text,             -- '리퀴시아' (NULL이면 경쟁사)
  ddd_qty integer,
  upload_id uuid FK → uploads.id
)

-- 업로드 이력
uploads (
  id uuid PK,
  uploaded_by uuid FK → auth.users.id,
  file_name text,
  uploaded_at timestamptz DEFAULT now(),
  row_count integer,
  status text  -- 'processing' | 'done' | 'error'
)
```

### 분석용 View (PostgreSQL)

```sql
-- 사업부별 성분시장 마켓쉐어
CREATE VIEW v_division_market_share AS
SELECT
  bm.division,
  m.year_month,
  m.ingredient_market,
  m.product_type,
  m.our_product,
  SUM(m.ddd_qty) AS total_ddd,
  SUM(m.ddd_qty) * 100.0
    / SUM(SUM(m.ddd_qty)) OVER (
        PARTITION BY bm.division, m.year_month, m.ingredient_market
      ) AS market_share_pct
FROM monthly_ddd m
JOIN brick_master bm USING (brick_code)
GROUP BY bm.division, m.year_month, m.ingredient_market,
         m.product_type, m.our_product;
```

## 각 페이지 상세 기획

### 로그인 (`/login`)
- Supabase `signInWithPassword` 호출
- 성공 시 `/` 리다이렉트

### 전체 현황 (`/`)
- 월 선택 필터 (Jan 2024 ~ Dec 2025)
- **자사 4제품 전체 DDD 합계** 카드 (리퀴시아 / 프리그렐 / 딜라트렌 / 베르쿠보)
- **사업부별 총 DDD** 비교 Bar Chart (8개 사업부)
- **전달 대비 성장률** 각 사업부 카드에 ▲▼ 표시
- 카드 클릭 → 사업부 대시보드

### 사업부 대시보드 (`/divisions/[division]`)
- 상단: 해당 사업부 성분시장별 **마켓쉐어** (Donut Chart 4개)
- 중단: **오리지널 vs 제네릭 비율** (Stacked Bar)
- 하단: **병원 목록 테이블** — 병원명 / 종합구분 / 총 DDD / 전달대비 성장률
- 병원 행 클릭 → 병원 대시보드

### 병원 대시보드 (`/divisions/[division]/hospitals/[brickCode]`)
- 성분시장 탭 전환 (해당 병원에서 자사 제품이 속한 시장)
- 탭별:
  - **마켓쉐어 추이** Line Chart (최근 12개월)
  - **당월 마켓쉐어 %** + **전달 대비 성장률 %** 강조 수치
  - **오리지널 vs 제네릭** Donut Chart
  - **제품별 DDD** Bar Chart (경쟁사 포함)

### 파일 업로드 (`/upload`)
- Drag & Drop Excel 파일 업로드
- raw 시트 + 브릭마스터 시트 자동 감지
- 업로드 전 미리보기 (첫 5행)
- 업로드 이력 목록

## Excel 업로드 처리 로직

```
1. xlsx 파일 업로드 (raw 시트 + 브릭마스터 시트 자동 감지)
2. wide → long format 변환
   Jan 2024, Feb 2024 ... 컬럼 → year_month 행으로 pivot
3. Pack Molecule String 파싱
   - " Gx" 포함 → product_type = 'generic', ingredient = Gx 제거한 값
   - 없으면 → product_type = 'original'
4. 제품 컬럼 비어있으면 경쟁사, 값 있으면 자사
5. Supabase DB upsert (brick_code + year_month + pack 복합 유니크)
6. 브릭마스터 시트 → brick_master 테이블 upsert
```

## 핵심 계산 로직

```
마켓쉐어(%) = 자사 제품 DDD / 해당 성분시장 전체 DDD × 100

전달대비 성장률(%) = (당월 DDD - 전월 DDD) / 전월 DDD × 100

오리지널 비율(%) = 오리지널 DDD / (오리지널 + 제네릭 DDD) × 100
```
